# Data Model: Project Analysis & Future Suggestions

## Core Entities

### AnalysisReport
**Purpose**: Root entity containing complete project analysis results

**Fields**:
- `id`: Integer, primary key
- `project_name`: String, name of analyzed project
- `analysis_date`: DateTime, when analysis was performed
- `version`: String, analysis version/iteration
- `status`: Enum (in_progress, completed, failed)
- `total_score`: Float, overall project health score (0-100)
- `execution_time`: Integer, analysis duration in seconds
- `codebase_size`: Integer, total lines of code analyzed
- `created_at`: DateTime
- `updated_at`: DateTime

**Relationships**:
- One-to-many with SuggestionItem
- One-to-many with AnalysisDimension
- One-to-many with MetricSnapshot

**Validation Rules**:
- `total_score` must be between 0-100
- `status` must be valid enum value
- `execution_time` must be positive
- `project_name` must be non-empty

### SuggestionItem
**Purpose**: Individual recommendation generated by analysis

**Fields**:
- `id`: Integer, primary key
- `analysis_report_id`: Integer, foreign key to AnalysisReport
- `title`: String, brief suggestion title
- `description`: Text, detailed explanation
- `category`: Enum (critical_fixes, technical_debt, feature_enhancements, architecture, testing_quality, documentation)
- `priority_score`: Float, calculated priority (0-100)
- `business_impact`: Integer, business value score (1-5)
- `technical_impact`: Integer, technical improvement score (1-5)
- `risk_level`: Integer, risk if not addressed (1-5)
- `implementation_effort`: Integer, effort required (1-5)
- `estimated_hours`: Integer, rough time estimate
- `affected_files`: JSON, list of files that would be changed
- `code_examples`: Text, relevant code snippets or examples
- `status`: Enum (pending, in_progress, completed, dismissed)
- `assigned_to`: String, optional assignee
- `notes`: Text, additional notes or comments
- `created_at`: DateTime
- `updated_at`: DateTime

**Relationships**:
- Many-to-one with AnalysisReport
- One-to-many with SuggestionTag

**Validation Rules**:
- All impact scores must be 1-5 range
- `priority_score` must be 0-100
- `estimated_hours` must be positive
- `category` must be valid enum value
- `status` must be valid enum value

### AnalysisDimension
**Purpose**: Stores scores and details for each analysis dimension

**Fields**:
- `id`: Integer, primary key
- `analysis_report_id`: Integer, foreign key to AnalysisReport
- `dimension_type`: Enum (code_quality, architecture, feature_completeness, technical_debt, user_experience, testing_coverage)
- `score`: Float, dimension score (0-100)
- `max_possible`: Float, maximum achievable score
- `details`: JSON, dimension-specific metrics and findings
- `improvement_potential`: Float, potential score increase
- `critical_issues`: Integer, count of critical issues found
- `created_at`: DateTime

**Relationships**:
- Many-to-one with AnalysisReport

**Validation Rules**:
- `score` must be between 0-100
- `max_possible` must be >= `score`
- `improvement_potential` must be non-negative
- `dimension_type` must be valid enum value

### MetricSnapshot
**Purpose**: Historical tracking of key project metrics

**Fields**:
- `id`: Integer, primary key
- `analysis_report_id`: Integer, foreign key to AnalysisReport
- `metric_name`: String, name of metric
- `metric_value`: Float, current value
- `previous_value`: Float, previous measurement (nullable)
- `change_percentage`: Float, percentage change from previous
- `trend`: Enum (improving, stable, declining)
- `measurement_unit`: String, unit of measurement
- `created_at`: DateTime

**Relationships**:
- Many-to-one with AnalysisReport

**Validation Rules**:
- `metric_name` must be non-empty
- `change_percentage` calculation must be accurate
- `trend` must be valid enum value

### SuggestionTag
**Purpose**: Flexible tagging system for categorizing suggestions

**Fields**:
- `id`: Integer, primary key
- `suggestion_item_id`: Integer, foreign key to SuggestionItem
- `tag_name`: String, tag label
- `tag_type`: Enum (technology, priority, area, custom)
- `created_at`: DateTime

**Relationships**:
- Many-to-one with SuggestionItem

**Validation Rules**:
- `tag_name` must be non-empty
- `tag_type` must be valid enum value

## Enums Definition

### AnalysisStatus
- `in_progress`: Analysis currently running
- `completed`: Analysis finished successfully
- `failed`: Analysis encountered errors

### SuggestionCategory
- `critical_fixes`: Security vulnerabilities, breaking issues
- `technical_debt`: Code refactoring, performance optimizations
- `feature_enhancements`: Missing functionality, UX improvements
- `architecture`: Design patterns, structure improvements
- `testing_quality`: Test coverage, quality assurance
- `documentation`: API docs, user guides, specifications

### SuggestionStatus
- `pending`: Not yet addressed
- `in_progress`: Currently being worked on
- `completed`: Implementation finished
- `dismissed`: Decision made not to implement

### DimensionType
- `code_quality`: Code complexity, maintainability, style
- `architecture`: Design patterns, structure, dependencies
- `feature_completeness`: Functionality gaps, user needs
- `technical_debt`: Performance, security, maintenance issues
- `user_experience`: UI/UX consistency, accessibility
- `testing_coverage`: Unit, integration, e2e test gaps

### TrendDirection
- `improving`: Metric getting better over time
- `stable`: Metric relatively unchanged
- `declining`: Metric getting worse over time

### TagType
- `technology`: Technology-specific tags (python, flask, etc.)
- `priority`: Priority-related tags (urgent, nice-to-have)
- `area`: Functional area tags (frontend, backend, database)
- `custom`: User-defined custom tags

## State Transitions

### AnalysisReport States
```
[Created] → in_progress → completed
                     ↘ failed
```

### SuggestionItem States
```
[Created] → pending → in_progress → completed
                 ↘ dismissed
```

## Relationships Summary

```
AnalysisReport (1) ←→ (N) SuggestionItem
AnalysisReport (1) ←→ (N) AnalysisDimension  
AnalysisReport (1) ←→ (N) MetricSnapshot
SuggestionItem (1) ←→ (N) SuggestionTag
```

## Storage Considerations

- **Primary Storage**: SQLite database (existing CRM database)
- **JSON Fields**: Use for flexible storage of analysis details, affected files, code examples
- **Indexing**: Create indexes on frequently queried fields (analysis_report_id, category, status, priority_score)
- **Retention**: Keep historical analysis reports for trend analysis
- **Backup**: Include in existing database backup strategy
